===REST===

1. AUTH
"Эндпоинт /push-console/api/v1/auth/password/change" (CHANGE PASSWORD)
Позитивные тест-кейсы:
- Проверки успешной попытки смены пароля с валидными данными

Негативные тест-кейсы:
- Проверка неуспешной попытки смены пароля с отсутствующим полем(new_password/old_password)
- Проверка неуспешной попытки смены пароля (старый пароль совпадает с новым)
- Проверка неуспешной попытки смены пароля при неавторизованном пользователе
- Проверка неуспешной попытки смены пароля при невалидном access_token

"Эндпоинт /push-console/api/v1/auth/logout" (LOGOUT)
Позитивные тест-кейсы:
- Проверка успешной попытки выхода с валидными данными (access_token)

Негативные тест-кейсы:
- Проверка неуспешной попытки выхода с пустым access_token
- Проверка успешной попытки выхода при невалидном access_token
- Проверка неуспешной попытки при двойном выходе

2. PROJECTS
"Эндпоинт /push-console/api/v1/projects/{id}" (DELETE)
Позитивные тест-кейсы:
- Проверка успешной попытки удаления проекта с валидными данными

Негативные тест-кейсы:
- Проверка неуспешной попытки удаления проекта с невалидным project_id
- Проверка неуспешной попытки удаления проекта при неавторизованном пользователе
- Проверка неуспешной попытки удаления проекта при невалидном access_token
- Проверка неуспешной попытки удаления уже удаленного проекта (двойное удаление)

"Эндпоинт /push-console/api/v1/projects/{id}" (UPDATE)
Позитивные тест-кейсы:
- Проверка успешной попытки обновления проекта с валидными данными

Негативные тест-кейсы:
- Проверка неуспешной попытки обновления проекта с невалидным project_name (request_body)
- Проверка неуспешной попытки обновления проекта при неавторизованном пользователе
- Проверка неуспешной попытки обновления проекта при невалидном access_token
- Проверка неуспешной попытки обновления проекта с невалидным project_id
- Проверка неуспешной попытки обновления проекта с отсутствующим полем "name"

"Эндпоинт /push-console/api/v1/projects/{id}/service-tokens" (ADD SERVICE TOKEN)
Позитивные тест-кейсы:
- Проверка успешной попытки добавления сервисного токена с валидными данными
- Проверка успешной попытки добавления нескольких сервисных токена с валидными данными

Негативные тест-кейсы:
- Проверка неуспешной попытки добавления сервисного токена при неавторизованном пользователе
- Проверка неуспешной попытки добавления сервисного токена при невалидном access_token
- Проверка неуспешной попытки добавления сервисного токена с невалидным project_id
- Проверка неуспешной попытки добавления сервисного токена с невалидным project_name

"Эндпоинт /push-console/api/v1/projects/{id}/service-tokens/{value}" (DELETE SERVICE TOKEN)
Позитивные тест-кейсы:
- Проверка успешной попытки удаления сервисного токена с валидными данными

Негативные тест-кейсы:
- Проверка неуспешной попытки удаления сервисного токена с невалидным значением

3. APPS
"Эндпоинт /push-console/api/v1/projects/{id}/service-tokens/{value}" (CREATE APP)
Позитивные тест-кейсы:
- Проверка успешной попытки создания приложения с валидными данными

Негативные тест-кейсы:
- Проверка неуспешной попытки создания приложения с отсутствующим полями
- Проверка неуспешной попытки создания приложения с невалидными app_name
- Проверка неуспешной попытки создания приложения с невалидным app_signature

"Эндпоинт /push-console/api/v1/projects/{id}/apps" (GET APPS)

Позитивные тест-кейсы:
- Проверка успешной попытки получения приложений проекта

Негативные тест-кейсы:
- Проверка неуспешной попытки получения приложений проекта с невалидным project_id
- Проверка неуспешной попытки получения приложений проекта при неавторизованном пользователе

"Эндпоинт /push-console/api/v1/projects/{project_id}/apps/{app_id}" (GET APP)
Позитивные тест-кейсы:
- Проверка успешной попытки получения определенного приложения

Негативные тест-кейсы:
- Проверка неуспешной попытки получения определенного приложения с несуществующим app_id
- Проверка успешной попытки получения определенного приложения при неавторизованном пользователе

"Эндпоинт /push-console/api/v1/projects/{project_id}/apps/{app_id}" (UPDATE APP)
Позитивные тест-кейсы:
- Проверка успешной попытки обновления приложения

Негативные тест-кейсы:
- Проверка неуспешной попытки обновления приложения при old_name = new_name
- Проверка неуспешной попытки обновления приложения с невалидным app_name
- Проверка неуспешной попытки обновления несуществующего приложения
- Проверка неуспешной попытки обновления приложения с невалидными project_id
- Проверка неуспешной попытки обновления приложения при неавторизованном пользователе
- Проверка неуспешной попытки обновления приложения при невалидном access_token
- Проверка неуспешной попытки обновления приложения с пустым request_body

"Эндпоинт /push-console/api/v1/projects/{project_id}/apps/{app_id}" (DELETE APP)
Позитивные тест-кейсы:
- Проверка успешной попытки удаления приложения

Негативные тест-кейсы:
- Проверка неуспешной попытки удаления несуществующего приложения
- Проверка неуспешной попытки удаления приложения с невалидным project_id
- Проверка неуспешной попытки удаления приложения при неавторизованном пользователе
- Проверка неуспешной попытки удаления приложения при невалидном access_token
- Проверка неуспешной попытки удаления уже удаленного приложения (двойное удаления)

4. USER
"Эндпоинт /push-console/api/v1/users/{user_id}/approve" (APPROVE USER)
Негативные тест-кейсы:
- Проверка неуспешной попытки одобрения заявки пользователя (не хватает прав)

"Эндпоинт /push-console/api/v1/users/{user_id}/approve" (BLOCK USER)
Негативные тест-кейсы:
- Проверка неуспешной попытки блокирования пользователя (не хватает прав)

"Эндпоинт /push-console/api/v1/users/{user_id}/promote" (PROMOTE USER)
Негативные тест-кейсы:
- Проверка неуспешной попытки продвигать пользователя (не хватает прав)
